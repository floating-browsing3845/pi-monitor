## 🚀 How It Works

pi-monitor uses a straightforward method to track your tasks. The software acts as a bridge between your active work and the background processes you run.

When you start a command, pi-monitor performs four tasks:

1. It triggers a background process based on your command.
2. It opens a floating terminal window on your screen. This window displays text as the process generates it.
3. It saves every piece of information to your local disk. This includes the process ID, the command you ran, the output logs, and the final exit status.
4. It alerts you when the task stops.

This approach ensures you never lose data, even if your machine restarts or the application closes.

## 📂 Understanding the Data Folders

pi-monitor keeps your computer tidy by organizing its work in one specific location. Every active task receives a unique ID. Under this ID, the software creates four files:

* pid: This file holds the process identification number.
* command: This file records the exact script or command you executed.
* output.log: This is your primary log file. It contains all text generated by your process.
* exitcode: This file appears only after the process ends. It tells you if the task finished successfully or if it encountered an error.

You can access these files anytime to review past work. They reside in your system temporary directory. This structure keeps your main work folders clean from clutter.

## 🧩 Frequent Questions

Can I run multiple tasks at once?
Yes. pi-monitor handles multiple background processes simultaneously. Each task receives its own floating window, which you can move or resize as needed.

Where does the software save my logs?
It saves logs in your temporary system storage folder under the pi-monitor directory. You can navigate there via your file explorer at any time.

How do I stop a process?
You stop a process by clicking the close button on the floating window. pi-monitor records an exit status for the task and saves the final log file immediately.

Does the application slow down my computer?
pi-monitor uses minimal system resources. It focuses on streaming text to the window and writing logs to the disk. It does not perform heavy calculations or data processing.

Will the logs grow too large?
The application writes text logs efficiently. For most users, these files consume very little space. You should clear your temporary folder periodically if you run hundreds of long-running tasks each week.

Does it work with other languages?
The monitor tracks any command that your Windows system can run. It treats all processes the same way, regardless of the tools or code used to create them.

## 📋 Troubleshooting

If you encounter issues during installation or usage, check these common areas first:

* Permissions: If the application cannot write logs, ensure you have sufficient permissions for the temporary folder.
* Path issues: Ensure that the command you wish to monitor is reachable by your system path. If the computer cannot find the command, pi-monitor will show an error in the floating window.
* Conflicts: If another application uses the same process ID, occasionally conflicts occur. pi-monitor generates unique IDs for every session to prevent this.

If these steps do not fix your issue, check the issues section on the main website to see if other users report similar problems. The project maintainers update the software to improve stability and performance based on user feedback.
